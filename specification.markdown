Разработать локальный скрипт, который загружает исторические данные по фьючерсам (Si, BR, NG) с таймфреймом 15 минут, идентифицирует заданные свечные паттерны за последние 2 торговых дня и визуализирует их на графике для проверки корректности.
2. Технологический стек

    Язык: Python 3.10+

    Источник данных: moexalgo (официальный API MOEX) или yfinance (как альтернатива, но для фьючерсов РФ лучше первый вариант).

    Анализ данных: pandas, TA-Lib (или pandas_ta) для автоматического определения паттернов.

    Визуализация: plotly (интерактивные графики) или mplfinance.

3. Список инструментов (Тикеры)
Инструмент	Код (пример)	Описание
Si	SiH6 (или актуальный)	Фьючерс на курс доллар-рубль
BR	BRH6	Фьючерс на нефть Brent
Gas	NGH6	Фьючерс на природный газ
4. Функциональные требования (Этап 1)
Модуль 1: Загрузка данных

    Скрипт должен запрашивать данные по OHLCV (Open, High, Low, Close, Volume).

    Интервал: 15 минут.

    Глубина истории: 2 дня (с учетом текущего момента).

Модуль 2: Детекция паттернов

На первом этапе ограничимся 3–5 классическими паттернами для теста:

    Молот (Hammer) / Перевернутый молот.

    Поглощение (Engulfing) — бычье и медвежье.

    Доджи (Doji).

    Утренняя/Вечерняя звезда.

Модуль 3: Визуализация и Тестирование

    Отрисовка японских свечей.

    Нанесение маркеров (стрелок или текста) над/под свечами, где обнаружен паттерн.

    Процедура теста: После запуска мы вручную сравниваем разметку алгоритма с графиком в TradingView или терминале брокера.

5. План разработки и итерации
Шаг 1: Настройка окружения и получение данных

    Установка библиотек.

    Написание функции get_data(ticker, timeframe).

    Тест 1: Вывод таблицы (Dataframe) с последними 5 свечами в консоль.

Шаг 2: Алгоритм поиска паттернов

    Подключение библиотеки анализа.

    Создание колонок в таблице для каждого паттерна (0 — нет, 100 — бычий, -100 — медвежий).

    Тест 2: Проверка через print, что в колонке паттернов появились значения, отличные от нуля.

Шаг 3: Построение графика

    Генерация HTML-файла с интерактивным графиком.

    Тест 3: Визуальный осмотр. Совпадает ли "Молот" на графике с реальной свечой с длинной тенью?

6. Дальнейшие этапы (Roadmap)

    Этап 2: Телеграм-оповещения. Создание бота, который работает в цикле и при появлении паттерна на закрытой 15-минутке присылает скриншот и текстовое сообщение.

    Этап 3: Фильтрация (Smart Filters).

        Тренд: Определение паттерна только по направлению EMA 200.

        Уровни: Детекция паттернов только вблизи уровней поддержки/сопротивления или зон ликвидности.

        Объем: Подтверждение паттерна всплеском вертикального объема.

    Этап 4: Статистика. Сбор базы данных: сколько паттернов отработали (цена прошла X пунктов в нужную сторону) за месяц.
